🗺️ System Overview
HOC OES is a connected dashboard system that gives every team — from compound to warehouse — a live view of what's happening on the floor.
What HOC OES is

It's a web-based operations system that runs on any tablet, phone, or computer with a browser. There's no software to install — you open a URL and you're in. Data flows between teams automatically: when the planner uploads tomorrow's schedule, every dashboard updates within seconds.

The core idea

Production data lives in one shared source of truth (the Supabase database). Every dashboard reads from it. Every action a user takes — marking a batch complete, flagging a quality issue, recording a changeover — writes back to it. Everyone sees the same numbers at the same time.

What HOC OES is NOT
Not...Because...
An ERP systemIt doesn't replace Vicinity, manage purchase orders, or do accounting
A WMSIt doesn't track every pallet location — FasTrak still does that
The OEE sourcePatch OEE is still the official OEE system; HOC OES displays Patch data, doesn't replace it
A replacement for the plannerThe planner still drives the schedule in Excel; HOC OES surfaces that schedule to the floor
The data flow (top to bottom)

Step 1 — Planner uploads. Every morning, the planner exports the Production Tracker from Excel and uploads it via the Data Upload Hub. This is the single source of truth for "what should run today."

Step 2 — Floor teams see their work. Compound sees what to mix, pre-weigh sees what to stage, QC sees what to test, production sees what to fill. Each dashboard filters the shared schedule to that team's responsibility.

Step 3 — Floor teams update status. Mark a batch as RUNNING, log a downtime, flag a quality hold, record a changeover. Each click writes to Supabase.

Step 4 — Leadership sees rollup. Control Tower aggregates everything: how many batches are running, how many are blocked, what risks exist, what OEE looks like across the plant.

Step 5 — Reforecast and feedback. If yield drops or capacity slips, Schedule Reforecast flags it back to the planner for the next day's adjustments.

Important: garbage in, garbage out If the planner uploads the wrong file, or floor teams forget to mark batches complete, the data goes stale fast. The system is only as good as the discipline behind it. We've added safety checks (staleness warnings, file validation, sanity checks on monthly numbers) but they don't replace good habits.

⚡ 5-Minute Quick Start
Get from "never used this" to "I know what to click" in under 5 minutes.
1
Open HOC OES on your device
Navigate to hocoperations1.github.io/Hoc-OES/. Bookmark it, or better — add to your tablet's home screen (see Install on Tablet).
2
Notice the freshness banner at the top
Green = data is fresh (uploaded today). Amber = data is more than 7 days old, you should re-upload. Red = no data has ever been uploaded. If you see amber or red, find the planner.
3
Open the dashboard for your role
From the index hub, find the dashboard that matches what you do. Compound = Compound Coordinator, fill operator = Production Supervisor, QC = Quality Lab, etc. See Role Guides.
4
Click on a KPI tile or batch row
Every number and row in HOC OES is clickable. Click an "Overdue" KPI to jump to Compound Coordinator. Click a RUNNING batch row to jump to Production Supervisor. The system routes you to the place where you can act on what you're looking at.
5
Look for the ? button
Every page has a blue ? button in the bottom-right corner. Tap it to bring back the Quick Start guide or jump to this onboarding tool. It's your safety net.

📋 Planner / Scheduler Guide
You're the data engine. Your morning upload drives every dashboard for every team. This guide covers your daily routine plus the troubleshooting essentials.
Your job is the most important one for HOC OES If your upload doesn't happen, the floor team sees stale data and trust in the system erodes within days. The system has fallbacks and warnings, but nothing replaces the daily upload.
Your daily 60-second routine
1
Update the Production Tracker in Excel
Do your normal planner work — update batch dates, statuses, quantities. Most importantly: keep Goal Source Data tab current with this month's actual cases (row 20) and OEE per line (rows 4-9 from Patch).
2
Save the file
File name doesn't matter, but it must be .xlsx format and contain the CLEAN_ DATA tab (note the space) and Goal Source Data tab.
3
Open HOC OES → Data Upload Hub
It's in the top action row (orange/red icon). On the desktop you can drag-and-drop; on tablet tap the upload zone.
4
Drop file in slot 1 (Production Tracker)
Watch the right-side progress log. You're looking for these green checkmarks:
• ✓ Parsed CLEAN_ DATA: 170+ rows
• ✓ Goals: actual=… goal=…
• ✓ Auto-detected XX changeovers
• ✓ Saved to Supabase
5
Verify on Production Supervisor
Click over to Production Supervisor and confirm the May Cases / Case Goal numbers match what you'd expect. If they don't, the file may have a column shift — see Troubleshooting.

⚙️ Shift Supervisor Guide
You're watching live production. Your primary dashboard is Production Supervisor — it shows what's running, what's stopped, and where you need to act.
Your shift start checklist (5 minutes)
1
Check the freshness banner
If amber/red, the planner hasn't uploaded — find them before you go any further.
2
Open Production Supervisor
Top row shows: May Cases, Case Balance, Open Batches, Running Now, Completed, OEE, Attainment, Case Goal. Memorize where each one lives — you'll glance at this row 50 times today.
3
Scan the Line Status table
Each line shows: total batches today, completed, running, pending, remaining minutes, available hours, gap. Red gap = behind schedule, green gap = ahead. Click a line row to drill into batch detail.
4
Review Capacity Forecast
Click "Capacity Forecast" in the nav. Shows next 14-30 days of batches. 1 MUST = overdue/running now. 4 SHOULD = scheduled this period. Load % over 100 = overcommitted.
5
Note any red KPIs and act
Click any red number — it routes you to the dashboard where you can fix it. Compound overdue? Goes to Compound Coordinator. Low yield? Goes to FG Reconciliation.
Common actions during shift
When you see...Do this
Batch RUNNING but compound not readyClick batch row → opens Compound Coordinator to expedite
Line down (no movement for 30+ min)Open Event Engine to log downtime
Major changeover happeningOpen Changeover Dashboard to track time
Yield looks offOpen FG Reconciliation to investigate batch-level variance
QC has a holdOpen Quality Lab to see what's blocked and why
Reading the KPIs honestly
OEE shows "—"? That means Patch data hasn't been uploaded today. It's NOT a bug — it's the system telling you it doesn't have data. Tell the planner.
May Cases looks too small or too big? The number comes from Goal Source Data row 20. If it's wrong, the source file is wrong. Don't trust the number until the source is verified.

📦 FG / Warehouse Guide
After production fills a batch, it lands in your area. Your dashboards are FG Reconciliation and Warehouse Movement.
FG Reconciliation

Compares what production says they made vs what you actually received. Variance = yield loss + counting errors + damages.

1
Review variance KPIs at top
Avg Yield, Low Yield count, Overfill count. Click any to drill into the flagged batches.
2
Verify counts on incoming pallets
Compare your physical count to the actual qty in Production Supervisor. Flag discrepancies in the batch row.
3
Mark short-shipped or damaged units
Adjusts the case balance for the month. Helps the planner understand true output.
Warehouse Movement

Tracks pallet movement between staging, QC hold, FG storage, and ship. Right now this is a planning view — FasTrak is still the official WMS until that integration is wired up.

🏭 Compound Manager Guide
Your job is matching batches to tanks based on capacity, capability, and category. Your primary dashboard is Compound Tank Board — it tracks all 14 mixing tanks and lets you assign verified batches.
This role is distinct from Compound Coordinator and Compound Scheduler Coordinator handles raw material verification. Scheduler handles formula sequencing. You handle which tank each batch goes into. Three roles, three different decisions, three different dashboards.
First-time setup (do this once)
1
Open Tank Config tab
From the top nav of Tank Board, click "Tank Config". You'll see all 14 tanks pre-loaded from the master spreadsheet with their physical specs (capacity, heated, double-motion sweep, homogenizer).
2
Set each tank's category
Categories: All-Purpose, Base, Cold Thinner, Oil & Thin, Gel, Other. Use the dropdown next to each tank. Per the master sheet, MT-18 is the typical Gel tank (homogenizer). MT-1 and MT-2 are usually All-Purpose (no heat). The rest depends on your operations.
3
Add notes (optional)
Notes field is for things like "requires special seal", "outside vendor cleaning required between gel batches", etc. Visible only on the Config tab — not on the main Tank Board.
Daily routine
1
Open Tank Board
From HOC-OES hub, click the teal "TB" card. Or direct URL: HOC_Tank_Board.html.
2
Scan the KPI strip
7 tiles at top: Total Tanks, Available, In Use, Cleaning, Offline, Pending Assign, Total Capacity. "Pending Assign" is the one to watch — if it's not zero, batches are waiting for you.
3
Click "Assign Batches" tab
Shows the list of batches awaiting a tank. Each row has Compound Coordinator verification status (Verified ✓ or Pending). Click "Assign tank →" to open the picker.
4
Pick a compatible tank
The picker shows ONLY tanks that are (a) currently Available status, and (b) large enough for the batch (approx 1 gal ≈ 8 lbs). Sorted smallest-fit-first. Each tank shows capacity + category + capabilities. Click "Assign →" to confirm.
5
Update tank status as work happens
Click any tank on the Tank Board to update: Filling (compound mixing now), Full (mixed, awaiting fill), Draining (being emptied to fill line), Cleaning (CIP in progress), Available (ready for next batch).
Tank capability rules

Capacity check is built in (won't let you assign a 3000-gal batch to MT-9). But other rules you should know:

CapabilityWhich tanksRequired for
HeatedMT-3 through MT-13, MT-18 (all except MT-1, MT-2)Most compounds that need temperature
Double-motion sweepMT-5 through MT-8, MT-10 through MT-13, MT-18Thick / viscous compounds
Homogenizer (built-in)MT-18 onlyGel batches (others need external homogenizer)
Tank status colors

The colored bar on the left of each tank card tells you status at a glance:

  • Green = Available (ready for next batch)
  • Blue = Filling (compound mixing now)
  • Purple = Full (mixed, waiting to be drained)
  • Amber = Draining (transferring to fill line)
  • Orange = Cleaning (CIP in progress)
  • Gray = Offline (out of service)
Dwell time alerts

When a tank is in Full or Draining status, the card shows "In tank: 2d 4h" — how long the batch has been sitting. Color-coded:

ColorDwell timeAction
White/normal< 24 hrOK
Amber1-3 daysCheck soon — material aging
Red> 3 daysInvestigate — may need re-test or re-work
Recent Activity tab

The Activity tab logs every status change and assignment for the last 50 actions. Useful for end-of-shift review or troubleshooting "what happened to this batch yesterday".

Tank assignments sync to other dashboards When you assign a batch to a tank, that info flows to Compound Coordinator (verification screen) and Compound Scheduler so the rest of the team knows where each batch lives. Don't bypass the dashboard — manual sticky notes break the sync.

📋 Production Coordinator Guide
Your job is making sure each scheduled batch has every component (caps, jars, labels, shippers) ready before the line needs them, and that materials get staged correctly. Your primary dashboard is Production Coordinator.
What this dashboard does for you Five workspaces: Command Center (open batches at a glance), Component Check (verify components for each batch), Inventory (live QOH for caps/jars/labels/shippers), Alerts & Escalations (multi-component shortage logging), and Staging Placard (new — generate printable LG-010-00 placards for material handlers).
Your daily routine
1
Open Production Coordinator
From index hub → amber "Production Coordinator" card. Or direct URL: HOC_Production_Coordinator.html.
2
Run Component Check on today's batches
Click "Component Check" tab. For each batch starting today/tomorrow, verify: caps · jars · labels · shippers · inner packs all have sufficient QOH. Mark ✓ if clear or log shortage if not.
3
Log multi-component shortages (if any)
Click "Alerts & Escalations" tab. Use the multi-shortage entry to add multiple low components for a single batch — select from dropdown of low-inventory items, enter qty needed, click + Add. Notifies procurement & planning automatically.
4
Generate Staging Placard for material handlers NEW
Click "Staging Placard" tab. Select a batch from the dropdown → click "Generate Placard". A printable LG-010-00 form pre-fills with Item Number, Batch Number, Line Number from the batch. Add up to 10 components: type/select SKU (autocompletes from warehouse inventory), enter Qty Needed, enter Pick Location (system suggests locations from past placards for this SKU). Click "🖨 Print Placard" — only the placard prints, not the rest of the dashboard. Material Handler then uses the FASCOR scan gun process to pull each item, signs off, hands placard back to Line Operator.
About the Staging Placard
Built from LG-010-00 Logistic Lines Staging Process SOP (issued Jan 21, 2026) The paper placard you used to fill in by hand is now generated. Material Handler runs the FASCOR scan gun process: Moveable Units → Inquiry Move → Date Inquiry → Scan SKU → Loc-to-Loc to line (e.g. IPRD010001). Both Material Handler and Operator initial each line; Supervisor initials when item count exceeds 3 pallets of jars/bottles or 20,000 labels.
Tips
Pick locations get smarter over timeFirst time you generate a placard for a SKU, type the location manually. Next time the same SKU appears, the system suggests it. After a few weeks, almost every SKU autocompletes.
Print only the placardThe print CSS hides nav, header, and all dashboard chrome — only the placard form prints. Works in Edge, Chrome, Safari.
Up to 10 items per placardPer SOP. If a batch needs more than 10 components, generate a second placard.
Audit trailEvery printed placard is saved to localStorage (last 100). To pull history, ask Claude to query hoc_placards_v1.
What this dashboard does NOT do (yet)
Auto-populate components from BOMYou enter the SKU + qty manually. BOM auto-fill is future work.
Track placard completion / sign-off digitallySignatures still happen on the printed paper. Digital signature capture is future work.
FASCOR scan-gun integrationMaterial handler still uses the scan gun manually per LG-010-00. We don't have a FASCOR API yet.
💼 Sales / Account Management Guide
Your job is keeping customers informed and ahead of issues. Your primary dashboard is Sales & Customer View — it translates production data into customer-facing terms.
What this dashboard does for you Takes the raw production schedule (compound, fill, QC, FG) and reorganizes it by customer instead of by line. So instead of "L10 has 15 batches running," you see "AJ has 8,400 cases planned this month, 60% fill rate so far, 3 orders at risk."
Your daily 5-minute routine
1
Open Sales & Customer View
From index hub → pink "Sales & Customer View" card. Or direct URL: HOC_Sales_Dashboard.html.
2
Scan the 7 KPI tiles at top
Active Customers, Cases Shipped, Cases Planned, Fill Rate, In Production, Completed, At-Risk Orders. The At-Risk number is the one you most want to be low. If it's red, you have proactive calls to make.
3
Review "Top Customers This Month"
Top 5 by case volume. Each row shows planned, shipped, fill %, at-risk count. Fill % under 80% = red — that's a customer who's frustrated. Click any row to filter the Orders tab to that customer.
4
Check "At-Risk Orders"
Shows which customers have orders in trouble. Click "View all risks" to see the full at-risk list with "what to tell the customer" text for each one.
5
Glance at the 6-month trend
Cases shipped per month, last 6 months. Flat or rising = good. Sharp drop = ask why (lost customer? capacity issue?).
Customer detail view

On the Customers tab, every customer has a card showing batches, planned cases, fill %, and at-risk count. Click any card to open the detail modal — full batch list for that customer with status pills and dates.

Use this when you're about to call a customer. "I see you have 8 batches scheduled this month, 5 are complete, 2 are running now, 1 is on QC hold for review — I'll have an update on that one by tomorrow."

Order pipeline

The Order Pipeline tab is your full order book for the month. Filter by customer, status, or month. Most useful for:

  • Pulling a customer's full order list before a meeting
  • Spotting orders that haven't started yet (Status = NOT STARTED)
  • Confirming what's ready to ship today (Status = COMPLETED)
At-risk orders

The At-Risk tab shows everything that's behind, on hold, or hasn't started yet. Each row has a "What To Tell Customer" hint:

StatusWhat to tell customer
COMPOUND OVERDUE"Production delayed; new ETA pending"
COMPOUND CRITICAL"On critical path; risk to ship date"
QC HOLD"Quality review in progress"
NOT STARTED"Scheduled but not yet started"
How customer is determined

HOC OES doesn't have an explicit "customer" field — it derives the customer from the batch description. So "AJ Quench Lv-In Cnd, 12 oz, 12ea" → customer = AJ (African Pride). "Hair Mayo 1.5oz packet" → customer = Hair Mayo.

If you see "Unknown" customers in the list, that means the description didn't match any of the known brand patterns. Tell the planner so we can add that brand to the recognition list.

📤 Daily Upload Guide
The single most important routine for HOC OES. Takes 60 seconds when it goes smoothly.
Who does it

The planner is the primary uploader. The shift lead is the backup if the planner is out. Anyone can do it in an emergency.

When to do it

Every morning by 7:00 AM. Also: any time the planner makes a significant schedule change during the day.

Step-by-step
1
Save the Production Tracker as .xlsx
If it's open in Excel, hit Ctrl+S. Make sure no other user has it locked.
2
Open Data Upload Hub
Either from the index hub (top action row, orange/red icon) or by direct URL.
3
Drag the file into slot 1 (Production Tracker)
On tablet: tap the slot, browse to the file. On desktop: drag-drop. Each slot has a colored border showing which file goes where.
4
Watch the progress log
Green checkmarks = good. Red errors = problem. The log tells you exactly what parsed, what didn't, and how many rows.
5
Verify on Production Supervisor
Confirm May Cases / Case Goal look right. If they're way off, the upload picked up wrong data — rollback and try again with the right file.
What "good" looks like
Successful upload checklist ✓ Green "Parsed CLEAN_ DATA: ~170 rows"
✓ Green "Goals: actual=85,097 goal=186,112"

🔄 Cross-Device Sync
When you make a change on one tablet, every other tablet sees it within 15-30 seconds. Here's how that works and how to verify.
How sync works

Every action (mark batch complete, log downtime, update QC) writes to Supabase. Every dashboard polls Supabase every 15 seconds for updates. So your change shows up on other tablets within one poll cycle.

How to verify it's working
1
Look for "LIVE · cross-device sync: 15s" indicator
Most dashboards show this in the header. Green dot = healthy. Red = sync down.
2
Test on two tablets at once
Tablet A: open Production Supervisor. Tablet B: open the same dashboard. On A, click a batch and make a change. Wait 30 seconds. B should reflect it.
3
If sync fails, check System Diagnostics
Go to Diagnostics → "Run All Checks". Look for the Supabase row.

🔧 Troubleshooting Guide
When something looks wrong, work through these checks in order.
Symptom: Dashboard shows "—" everywhere

Most likely: No upload has happened, or upload failed.

Fix: Open Data Upload Hub. Look at the freshness line. If it says "—" or "never", do an upload. If it says a date from days ago, do a fresh upload.

Symptom: Numbers look way off (too small or too big)

Most likely: Wrong file uploaded, or column shift in Excel.

Fix: Open the Production Tracker Excel. Verify the CLEAN_ DATA tab has standard columns (Date, Line, Batch, Item, Description, etc.). Re-save and re-upload.

Symptom: One tablet works, another doesn't

Most likely: Cache issue on the broken tablet, or the broken tablet has no internet.

Fix:

1. Check tablet's WiFi is connected
2. Hard refresh: pull down on the page in Chrome to refresh
3. Clear cache: Chrome → Settings → Site settings → hocoperations1.github.io → Clear data
4. Re-open the URL fresh

Symptom: Sync delay is over a minute

Most likely: Network issue, or Supabase free tier rate-limit.

Fix: Check System Diagnostics → Supabase row. If green, give it another minute. If red, escalate.

Symptom: Dashboard won't load at all

Most likely: Service worker cache is corrupt, or GitHub Pages is having an outage.

Fix:

1. Check if other websites work (test wifi)
2. Try opening hocoperations1.github.io/Hoc-OES/ in a different browser
3. If still broken, the issue is on our side — wait 10 minutes, try again
4. If still broken after 10 min, contact the System Admin

Symptom: PWA install option missing

Most likely: Wrong browser or wrong device mode.

Fix: Make sure you're using Chrome (Android) or Safari (iPad), not an in-app browser like the Outlook app's web view. Open the URL in a real browser.

📖 Glossary
Every acronym and term in HOC OES, defined plainly.
Attainment
% of planned cases that were actually produced. Goal: 90%. Measured from Goal Source Data.
Batch
A single production run of one item. Each batch has a unique MAIN###### batch number.
Case
A case of finished product (e.g., 12ea bottles). The standard unit HOC plans by.
Case pack
How many eaches are in one case (e.g., 12). Used to convert cases ↔ eaches.
Changeover
The time between two different batches on the same line. Formula change = 90 min, size change = 60 min, pack change = 30 min.
Compound
The pre-fill mixing step. Compound team makes the bulk product before fill team puts it in containers.
Eaches
Individual units. 12 eaches per case (typically). What customers see on the shelf.
FG
Finished Goods. Product that's filled, sealed, palletized, and ready to ship.
Formula
The recipe ID for what's being made (e.g., F.698 = AJ Knot On My Watch Detangler).
Item
The SKU code (e.g., 1-698-12-1200 = formula 698, 12oz, 12-pack, product ID 1200).
KPI
Key Performance Indicator. The numbers shown at the top of each dashboard.
Line
A production line. HOC has lines 1-5, 8, 10, 11, 14, 15, 16.
OEE
Overall Equipment Effectiveness = Availability × Performance × Quality. Goal: 60%. Comes from Patch.
Origqty
Original Quantity. How many cases the planner intended to make.
Patch
Patch Production Monitoring — the OEE system that monitors fill line speed and downtime.
Pre-Weigh
The step before compound where raw materials are measured out per batch recipe.
PWA
Progressive Web App. A website that installs like an app and works offline.
QC
Quality Control. Tests viscosity, density, pH, etc. before fill is approved.
Reforecast
Adjusting the schedule when yield or capacity changes. Triggered by Schedule Reforecast dashboard.
RLS
Row Level Security. Supabase's permission system. HOC OES has it set to "open" (anyone with the URL can read/write).
Service Worker
A browser tech that caches HOC OES files locally so it works offline. Updates whenever the cache version bumps.
Supabase
The cloud database HOC OES uses. Stores schedule, batches, sync events. Free tier currently.
Vicinity
HOC's ERP. The source of compound standards and fill labor data.
WMS
Warehouse Management System. HOC uses FasTrak — not yet integrated with HOC OES.
Yield
Actual cases ÷ planned cases. 100% = exactly hit plan. <95% = underfill. >102% = overfill.
